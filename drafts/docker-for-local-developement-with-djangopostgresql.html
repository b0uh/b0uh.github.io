Docker for local developement with Django/PostgreSQL

  Tue 02 May 2017

Introduction

Goal

The aim of this short tutorial is to run an existing (or new) Django project with PostgreSQL on your computer without having to install anything but Docker. In a first time will see how to run django within Docker, then we will add the postgresql management and finally load a DB dump. According to your need you may not need to follow all steps.

Requierements

Well, they are quite simples :

  • Docker (installation)
  • Docker Compose (installation)
  • A Django project
  • A PG dump made with pg_dump

Step 1 : Django

Here we will see how to run Django with sqlite.

Inside your Django project add a file docker-compose.yml with :

version: '2'
services:
  web:
    build: .
    command: python manage.py runserver 0.0.0.0:8000
    volumes:
      - .:/code
    ports:
      - "8000:8000"

And another file Dockerfile with :

FROM python:3.6
ENV PYTHONUNBUFFERED 1
RUN mkdir /code

WORKDIR /code
ADD requirements.txt /code/
RUN pip install -r requirements.txt
ADD . /code

Then run in your shell :

docker-compose up

It will build the "web" service the first time so it might take a while. At some point you should see :

web_1  | Performing system checks...
web_1  |
web_1  | System check identified no issues (0 silenced).

web_1  | You have 13 unapplied migration(s). Your project may not work properly until you apply the migrations for app(s): admin, auth, contenttypes, sessions.

Django is running ! Go to http://localhost:8000/, you should see it running :) You can stop it with CTRL+C.

At this point you can change your code, Django will reload automaticaly and we will see your changes.

If you need to changes requierements.txt you will need to rebuild the service, you can do with :

docker-compose up --build

If you need to run a command, you can do :

docker-compose run web python manage.py migrate

After this step you can run your Django with sqlite and change the code easily.

Step 2 : Django + PostgreSQL

Update of docker-compose.yml

version: '2'
services:
  db:
    image: postgres
    volumes:
      - ./docker/postgres:/var/lib/postgresql/data
  web:
    build:
      context: .
      dockerfile: ./docker/Dockerfile
    command: python manage.py runserver 0.0.0.0:8000
    volumes:
      - .:/code
    ports:
      - "8000:8000"
    depends_on:
      - db

In order to keep the project clean I choose to create a docker folder were we will put the Dockerfile unchanged and postgreSQL datas.

You can update your settings in order to use the new DB in your settings.py :

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql_psycopg2',
        'NAME': 'postgres',
        'USER': 'postgres',
        'PASSWORD': 'postgres',
        'HOST': 'db',
    }
}

The HOST in the service name from docker-compose.yml. The USER and PASSWORD are the default of the postgres Dockerfile.

And you got a Django/postgres project running with persistent datas on your harddrive ! :)

Step 3 : Django + PostgreSQL + SQL dump

Update our docker-compose.yml:

version: '2'
services:
  db:
    build:
      context: .
      dockerfile: ./docker/Dockerfile-db
    volumes:
      - ./docker/postgres:/var/lib/postgresql/data
  web:
    build:
      context: .
      dockerfile: ./docker/Dockerfile-web
    command: python manage.py runserver 0.0.0.0:8000
    volumes:
      - .:/code
    ports:
      - "8000:8000"
    depends_on:
      - db

You can see that I rename Dockerfile to Dockerfile-web because we now have another Dockerfile for the db service called Dockerfile-db.

Dockerfile-db

FROM postgres:latest
WORKDIR /docker-entrypoint-initdb.d
ADD docker/1-init-db.sh /docker-entrypoint-initdb.d/
ADD docker/2-load-dump.sh /docker-entrypoint-initdb.d/
ADD docker/my_project.dump /tmp

The goal of this Dockerfile is to initiate the DB with somes scripts and the dump.

Django advanced settings

TL;DR

Use the DJANGO_SETTINGS_MODULE variable to choose your Django settings according to your environnement and use a bit of code to load settings from your applications.

Cathing up with Django settings

Django settings are great because there are quite simple. You put anything in your settings.py file and then you can easily use them in your project.

from django.conf import settings

print(settings.LANGUAGE_CODE)

That's nice, but on large project you may encounter two issues:

  • How handle several environnements ? (production/stage/test)
  • Your settings.py may be overloaded by settings used only by specific applications. How dispatche those settings ?

Handle several environnements

With large project often come multiples environnements, for each on them you may want differents settings. For example you may want to active DEBUG in the development environnement and deactivate it in stage and production. Currently, if you start a new Django project, it should like this :

project
│   manage.py
│
└─── mysite
    │   __init__.py
    │   settings.py
    │   urls.py
    │   wsgi.py

The idea is to do :

project
│   manage.py
│
└─── mysite
    │   __init__.py
    │   urls.py
    │   wsgi.py
    └───settings
        │   base.py
        │   production.py
        │   stage.py
        │   development.py

django-registration vs django-registration-redux

Thomas Loiret
  Tue 11 July 2017


TL;DR: Use django-registration-redux because the community is behind it.


I have been a bit cut off the rest of the Django's world these last years and if you have been too, you may have some difficulties to understand why there is django-registration-redux and django-registration.

So I wanted to share some clues I got. I may have missed some!

The package django-registration use to be quite useful some years ago, but it seems like James Bennett stop maintains the package for more than 2 years (2013/2014), so Andrew Cutler forks it in 2014 and rename it django-registration-redux.

A year latter James Bennett start commiting again on the django-registration package. So now we have those two packages with differents names but the same application name, it can be quite disturbing when switching projects with those two packages.

Both are now actives and seems to provide the same features, but I have chosen django-registration-redux because the community seems to be behind it. One way to see it is to take a look at the statistics:

  • django-registration-redux
    • 63 contributors
    • 125 pull requests
  • django-registration
    • 21 contributors
    • 40 pull requests

I don't know why those projects are not merged into one. I think it could be really a good thing for the community and for the maintainers of those packages to join forces to have one package :)

Anyway, thanks to James Bennett, Andrew Cutler and all contributors to provide us such good packages!
